## 2. 论文如何评价“换模型”

### 2.1 总体结论
论文结论是：
1. MeshAgent 在不同模型上都能提升（跨 GPT-4o / Gemini / DeepSeek）。
2. 约束与验证流程比“单纯换更复杂模型”更关键。

### 2.2 关键结果（Table 4, reliable accuracy）
以 CoT+Few-shot + MeshAgent 为例：
1. TA 应用
   - GPT-4o: 0.987
   - Gemini-2/1.5: 0.956
   - DeepSeek-v3: 0.962
2. MALT 应用
   - GPT-4o: 0.986
   - Gemini-2/1.5: 0.958
   - DeepSeek-v3: 0.964
3. CRG 应用
   - GPT-4o: 1.000
   - Gemini-2/1.5: 0.957
   - DeepSeek-v3: 0.989

作者同时指出：搭配 MeshAgent 时，Fine-tuned 版本通常比 Few-shot 更稳定。

## 4. 论文中其他模型怎么落地

### 4.1 Gemini 路线
1. 更接近 Google API / Vertex 体系，不是 Azure OpenAI 原生路径。
2. 这仓库虽然有 Vertex 相关痕迹，但主流程默认是 Azure Search + Azure OpenAI。
3. 预期工作量：中到高（需要切 provider 及验证完整链路）。

### 4.2 DeepSeek-V3 路线
1. 不属于 Azure OpenAI 原生 GPT deployment 体系。
2. 需要走其他推理接入方式（如对应平台 API / 托管服务）。
3. 预期工作量：高（SDK、鉴权、输出格式、重试与日志都要适配）。
